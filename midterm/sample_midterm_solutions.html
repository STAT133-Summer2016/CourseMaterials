  }
  return(g)
}

h(5, g, f)
  1. What is the value of x after the following line of code is run?
x <- sapply(1:5, function(t) h(t, 0, 1))

Solution

Part (a)

## [1] 3

Part (b)

## [1] 0 1 1 2 3

Part (c)

  y[t] <- h(t, 0, 1)

Sample Question 6

What is the output of the code below? Explain. If it would return an error when run, write ERROR and the reason. You dont need to state the whole error message.

Part A

x <- "-"
str1 <- c("a","b","c")
str2 <- "z"
str_c(str1, str2, sep = "x")

Part B

x <- 1:3
if (x > 1) {
  x + 1
}

Part C

c(factor("z"), 1 < 3, 0) + TRUE

Solution Part A

"axc" "bxc" "cxc"

"a", "b", and "c" are pasted to "z" with "x" as a separator. The sep argument is "x" the string and not x the variable.

Part B

ERROR. If can only handle logical vectors of length 1. x > 1 would produce c(F, T, T), which has length 3.

Part C

2 2 1

Since vectors are atomic objects, R will implicitly coerce the data type to the most compatible type. Factors are stored as integers under the hood, so since "z" has one level, it is understood as 1. 1 < 3 is FALSE, which can be coerced to 0. The whole vector, then, takes on the numeric type. Attempting to add TRUE will coerce it to being 1.

Sample Question 9

You come across a function that youve never seen before, sobad. Like a sensible person, you look up the documentation to figure out how it works, and this is what you get:

sobad {terrible.package}

Description
Does some mathy things according to some stuff

Usage
sobad(poop, wow = NULL, ...)

Arguments
poop - a data frame
wow - a logical vector.  If TRUE, a sum will be taken.  Otherwise, the function will return the product.
... - strings with digits specifying the rows to apply the specified operation to

Value
A list, row-wise summaries of the data (only sums and products currently supported)

Examples
prod <- "2"
df <- data.frame(a = c(1,3), b = 1:2)
# This returns 6
sobad(df, F, prod)

You have the following objects:

alpha <- 1
gamma <- 3
a <- c(alpha, gamma)
b <- 4:6
lala <- 8 > 1
x <- data.frame(a = 1:3, b = 4:6)
y <- list(a = 1:3, b = 4:6)
wow1 <- function(x) sum(x)
wow2 <- function(x) prod(x)
wow3 <- function(x) as.character(x)
fafa <- "sum"

Using only the objects defined above, construct a function call to sobad that will return a length 2 numerical vector with the values 5 and 9. Explain your answer.

Solution

sobad(x, lala, wow3(alpha), wow3(gamma))

x is the only data frame we have to work with. lala evaluates to the logical value TRUE, which makes sobad take rowwise sums. To get 5 and 9, we need to add the 1st and 3rd rows together. To specify that, we need the strings "1" and "3", which arent direct options, but wow3(alpha) and wow3(gamma) will return what we need.
